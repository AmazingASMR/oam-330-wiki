# How this wiki is organized

A quick orientation to the wiki's structure and conventions, so you can find what you need without clicking through every page.

## Folders and their purpose

```
/                  top-level pages
  index            full catalog of every page
  coverage         every concept grouped by module
  about            project background

/modules           one page per module (M1–M12)
/concepts          132 short pages, one per term
/readings          22 reading summaries
/cross-cuts        synthesis pages spanning multiple modules
/practice          self-test question sets with answer keys
```

## Page conventions

Every page of the same type follows the same structure. This makes it easy to skim across pages without re-orienting each time.

**Concept pages** — one short page per term, capped at ~250 words.
- Frontmatter: `term`, `module`, `sources`
- Body: Definition → Why it matters → Key distinctions → Example(s) → Notes on how the idea is typically applied or tested

**Module pages** — a one-page overview per module.
- "What this module is fundamentally about" (3–5 sentences)
- Concept list grouped thematically, with a one-line gloss on each link
- Assigned readings for the module
- Cross-cut connections to other modules

**Reading pages** — one per assigned reading.
- Source → Main argument → Key terms introduced → Examples used → Connections to lecture concepts
- Source PDFs and slides are not redistributed; pages are summaries only.

**Cross-cut pages** — synthesis pages on themes that recur across multiple modules (cultural differences, fairness/justice, System 1 vs. System 2, decision-making biases, distractor pairs).

**Practice pages** — short self-test question sets per module, with answer keys and per-distractor explanations. Useful for self-checking your understanding.

## Navigation tips

- **Looking for a specific concept?** Open [[index|the concept index]] and Ctrl-F.
- **Refreshing a whole module?** Start with the module page (e.g. [[modules/03-decision-making]]). It links every concept in that module with a one-line gloss.
- **Want to see how ideas connect across modules?** Read the cross-cut pages. The [[cross-cuts/distractor-pairs|distractor pairs]] page in particular consolidates close-pair distinctions that recur throughout the course.
- **Testing yourself?** The practice sets in [[index#concept-pages-by-module-full-list|/practice]] are short and module-scoped; each question has explanations for why each option is or isn't correct.
